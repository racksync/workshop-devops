แอปทดสอบช่องโหว่ OWASP

แอปพลิเคชันนี้มีช่องโหว่ความปลอดภัยตามรายการ OWASP Top 10 เพื่อศึกษาและเรียนรู้

1. ช่องโหว่ SQL Injection

ลองใช้ admin' -- เป็นชื่อผู้ใช้ และรหัสผ่านอะไรก็ได้

2. ช่องโหว่ Cross-Site Scripting (XSS)

ลองใส่ <script>alert('XSS')</script> ในช่องข้อความ

ข้อความทั้งหมด

3. ช่องโหว่ Insecure Direct Object References (IDOR)

ลองป้อน ID ข้อความต่างๆ เพื่อเข้าถึงข้อความที่คุณอาจไม่มีสิทธิ์

4. ช่องโหว่ Broken Access Control

ลองเข้าถึงข้อมูลผู้ใช้ทั้งหมด (สำหรับแอดมินเท่านั้น)

5. ช่องโหว่ Security Misconfiguration

ตรวจสอบข้อมูลการกำหนดค่าเซิร์ฟเวอร์ที่รั่วไหลออกมา

6. ช่องโหว่ Sensitive Data Exposure

ตัวอย่างการเก็บข้อมูลที่ละเอียดอ่อนไว้ในเว็บแอปพลิเคชัน

7. ช่องโหว่ Cross-Site Request Forgery (CSRF)

ตัวอย่างแบบฟอร์มที่ไม่มีการป้องกัน CSRF

ลองดูการโจมตีผ่าน CSRF:

8. ช่องโหว่ Using Components with Known Vulnerabilities

แสดงรายการไลบรารีที่มีช่องโหว่ที่ใช้ในแอปพลิเคชัน

9. ช่องโหว่ Insufficient Logging & Monitoring

ทดสอบความล้มเหลวในการเข้าสู่ระบบหลายครั้งโดยไม่มีการล็อกที่เหมาะสม

10. ช่องโหว่ XML External Entities (XXE)

ลองโจมตี XXE โดยการส่ง XML ที่มี DOCTYPE ไปยังเซิร์ฟเวอร์
